HTML Webpage Automation with Python

5.17.2020

Hosting this site on GitHub Pages means I can't do anything backend like PHP. This presents an interesting problem of how to maintain static HTML pages. It can take a long time to manually edit HTML pages, especially if the same information is being edited in many different places. I created an automated process with Python for generatng HTML content pages.

Let's say I want a page to show off some photos. As a starting point, I could fill a folder with images that I want to share:

With Python, it's easy to iterate through the images:

    img_arr = sorted(glob.glob(pattern), reverse=True)

I also want to know when the image was created ('Datetime') or any captions ('UserComment') that might have been set. This can be contained in the image's EXIF metadata. Images should usually have 'DateTime' metadata but other metadata like "UserComment" can be set with photo managment software. The simple example code below extracts UserComment data:


    # the Pillow (PIL) and piexif libraries
    # worth mentioning, very helpful
    from PIL import Image
    import piexif
    # [...]
    im = Image.open(img_dir)
    exif_dict = piexif.load(im.info.get("exif"))
    user_comment = exif_dict["Exif"].get(piexif.ExifIFD.UserComment)

Once the image data is extracted, it can be exported into a CSV.

With that data in place, it's simple to write that data to some HTML templates. Example code:


    html_image_entry_temp = """
    <tr class="content">
	    <td class="date"><time>date_text</time></td>
	    <td class="image"><a href="img_src"><img src="thumbnail_src"></a><p>caption</p></td>
    </tr>
    """
    image_mid = ""

    # [...]

    image_entry = image_entry.replace("caption", caption)

The final result only has two steps: (1) set images in the folder with correct EXIF metadata, and (2) run the program. Pretty easy.
